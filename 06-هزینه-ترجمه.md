# هزینه ترجمه دیتاست به فارسی

---

## چه چیزی باید ترجمه شود؟

<div dir="rtl">

سرور محیط سه فایل را برای هر زبان لود می‌کند (`app_multi.py` خط ۴۷۸–۴۸۰). پس نسخه فارسی به سه فایل جدید `_fa.json` نیاز دارد. حجم و اهمیت‌شان زمین تا آسمان فرق دارد.

</div>

### الف) دستورالعمل‌ها (goals) — `items_human_ins_fa.json`

<div dir="rtl">

اعداد واقعی از فایل انگلیسی:
- ۴۹۸ محصول (ASIN) که با مقاله می‌خواند (≈۵۰۰ دستور انتخاب‌شده از ReAct)
- ۸۱۰ دستور در مجموع
- ۶۱ هزار کاراکتر ≈ **۱۵ هزار توکن**

این تقریباً هیچی است. کل دستورات را می‌توان با چند سنت ترجمه کرد.

</div>

### ب) محصولات (environment) — `items_shuffle_40k_fa.json`

<div dir="rtl">

اینجا غول واقعی است:
- ۴۲٬۱۳۹ محصول (دقیقاً مطابق مقاله: ۴۲٬۱۳۹ × ۱۴ زبان = ۵۸۹٬۹۴۶)
- متن قابل‌ترجمه (name, full/small_description, category, query): **۸۵.۵ میلیون کاراکتر**
- ≈ **۲۱.۴ میلیون توکن ورودی**
- فیلدهایی که نباید ترجمه شوند (ASIN، URL، قیمت، تصویر، seller_id) دست‌نخورده کپی می‌شوند.

</div>

### ج) attributeها — `items_ins_v2_40k_fa.json` (~15MB)

<div dir="rtl">

این فایل را سرور برای تب «ویژگی‌ها»ی صفحه محصول استفاده می‌کند. برای یک محیط کامل باید ترجمه شود، ولی چون از همان متن محصولات استخراج شده، می‌توان دوباره با TF-IDF تولید کرد (هرچند TF-IDF برای انگلیسی تنظیم شده؛ ترجمه ساده‌تر است).

</div>

## راهکارهای رایگان و ارزان

### ۱. مدل‌های ترجمه محلی (رایگان، بدون API)

```python
# Helsinki-NLP/opus-mt-en-fa روی HuggingFace:
from transformers import MarianMTModel, MarianTokenizer

model_name = "Helsinki-NLP/opus-mt-en-fa"
tokenizer = MarianTokenizer.from_pretrained(model_name)
model = MarianMTModel.from_pretrained(model_name)

def translate(text):
    inputs = tokenizer([text], return_tensors="pt", padding=True)
    output = model.generate(**inputs)
    return tokenizer.decode(output[0], skip_special_tokens=True)
```

### ۲. Meta NLLB-200 (کیفیت بالاتر از MarianMT)

```python
from transformers import pipeline

translator = pipeline(
    "translation",
    model="facebook/nllb-200-distilled-600M",
    src_lang="eng_Latn",
    tgt_lang="pes_Arab"  # فارسی
)
```

### ۳. Google Translate رایگان (غیررسمی)

```python
from deep_translator import GoogleTranslator
result = GoogleTranslator(source='en', target='fa').translate("hello world")
```

## پیشنهاد عملی

<div dir="rtl">

**گام اول:** با NLLB-200 شروع کن. چند نمونه از `items_shuffle` را ترجمه کن و کیفیت را بررسی کن.
- اگر برای benchmark قابل قبول بود: تمام دیتاست را رایگان ترجمه می‌کنی.
- اگر کیفیت کافی نبود: فقط برای بخش `items_human_ins` (که ۸۱۰ دستورالعمل کوچک است) از Haiku استفاده کن — هزینه‌اش زیر $1 می‌شود.

**گام دوم (محصولات):** GPT-4o از طریق Batch API:
- نصف هزینه (~$175)، کیفیت GPT-4-کلاس
- فقط فیلدهای متنی را در prompt بفرست، JSON structure را دست نزن
- می‌توان چند محصول را در یک درخواست batch کرد تا سربار prompt کم شود

</div>

## نکات فنی حیاتی برای فارسی

<div dir="rtl">

- **RTL:** فارسی راست‌به‌چپ است؛ باید `'fa'` را به لیست `languages` در `app_multi.py:475` اضافه کنی و رندر template را بررسی کنی.
- **پرامپت‌ها:** در `WebShop_prompt.py` کلمات اکشن (search/click) چندزبانه‌اند؛ باید معادل فارسی اضافه شود.
- **موتور جستجو:** `init_search_engine` برای هر زبان جداگانه ساخته می‌شود؛ توکنایزیشن فارسی (نیم‌فاصله، عدم وجود فاصله مشخص) ممکن است نیاز به تنظیم داشته باشد.
- **قیمت:** تبدیل قیمت به ریال باید در بومی‌سازی اضافی اعمال شود.
